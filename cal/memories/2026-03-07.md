## Meeting: Interactive Feature (v3.0 Vision)

**Date:** 2026-03-07
**Participants:** steve (product), swift-swiftui-expert (technical), Cal (facilitator)

### Key Insights

- The .md file is the shared artifact between human and AI. The markdown IS the protocol.
- The interaction model is "Respond, don't edit" — the document is in control, the user acts on specific affordances the AI left for them.
- Push notifications with deep links close the collaboration loop: AI writes, Pixley notifies, human responds, AI sees changes.
- The app evolves from passive reader to asynchronous collaboration layer between human and AI, mediated by the filesystem.
- One unifying rule: checkboxes outside blockquotes = multi-select, inside blockquotes = radio. This eliminates the need for separate yes/no vs multiple-choice patterns.

### Decisions Made

- **Five interactive patterns (not six) in scope for v3.0** — Checkboxes, Choices (merged yes/no + multiple choice), Fill-in-the-blank (with file/folder picker), Feedback, Approvals. Rationale: yes/no and multiple choice are the same mechanic (radio in blockquote), merging simplifies detection. Confidence: high.
- **File/folder picker via `[[choose file]]` / `[[choose folder]]`** — Extends fill-in-the-blank with type hints. Triggers NSOpenPanel, writes selected path back to markdown. Gives AI access to real filesystem paths the human selected. Rationale: AI often needs file paths; this is safer and more usable than asking humans to type paths. Confidence: high.
- **Push notifications with deep links** — Notify humans when AI changes watched files. User-configurable trigger rules (folder, file pattern, content type). Rationale: closes the human-AI collaboration loop. Confidence: high.
- **"Respond, don't edit" framing** — The app stays a reader with structured interaction points. Not a markdown editor. Rationale: different product with clear identity. Confidence: high. Revisit-if: users consistently ask for full editing.
- **Setup Files (renamed from "Sample Files") include a copy-paste AI prompt** — Users paste it into ChatGPT/Claude/Cursor to teach the AI the Pixley interactive markdown protocol. Rationale: defines the protocol AND onboards users in one move. The prompt IS the spec. Confidence: high.
- **The prompt template defines the protocol** — Whatever patterns we put in the prompt are the patterns we detect. Design the prompt first, implement detection second.
- **Four additional patterns added from community research** — CriticMarkup (inline accept/reject suggestions), Status state machines (draft→review→approved), Confidence indicators (high/medium/low on AI recommendations), Conditional/collapsible sections (show/hide based on choices). Plus auto-calculated progress bars per section (rendering only). Rationale: CriticMarkup enables live updates without reload; status machines give documents lifecycle; confidence helps humans focus review effort; conditionals keep multi-path documents clean. Confidence: high.
- **AI chat as command layer for document edits** — Users can say "I QA'd phase 1, it all passed" in the chat and Foundation Models translates that to bulk edits on interactive elements. Uses existing FM tool-calling (`@Generable` + `@Tool`) with an `editInteractiveElements` tool. AI asks follow-up for ambiguous commands ("all but 3 passed" → "which 3?"). Rationale: clicking 20 checkboxes one by one is tedious; natural language is faster for bulk operations. Confidence: high. Revisit-if: FM tool-calling proves unreliable for this use case.

### Technical Architecture (from swift-swiftui-expert analysis)

- **Click detection:** `mouseDown` override in `MarkdownNSTextView` subclass. Detect checkbox/pattern ranges via regex, toggle state, write back to file.
- **File write-back:** Read-modify-write from DISK (never from textStorage). Use `atomically: true`. Critical safety rule.
- **Instant feedback:** Update `DocumentState.content` in-memory after successful write. Suppress FileWatcher reload pill for self-initiated changes.
- **Fill-in-the-blank / comments:** Overlay `NSTextField` at computed text positions, or use popover/ChatView pre-fill as simpler alternative.
- **File/folder picker:** `[[choose file]]` → `NSOpenPanel(canChooseFiles: true)`, `[[choose folder]]` → `NSOpenPanel(canChooseDirectories: true)`. Selected path written back as plain text. NSOpenPanel grants temporary sandbox access.
- **Notifications:** `UNUserNotificationCenter` local notifications. FolderWatcher already detects changes. Deep link via `BrowserOpenRequest` + `WindowRouter`.
- **Performance:** Full re-highlight after toggle is fast enough (<50ms on Apple Silicon). Incremental re-highlight is a premature optimization.
- **Detection priority:** Approvals first (special case of blockquote checkbox), then Choices (remaining blockquote checkboxes), then Checkboxes (outside blockquotes), then Fill-in-the-blank, then Feedback.

### Research Findings (Reddit/HN/Dev Community)

**Key validation:** Nobody has built a markdown reader that treats markdown as a bidirectional collaboration surface between AI and humans. Pixley occupies genuinely uncharted territory. Closest precedents are Obsidian's checkbox renderer plugin and CriticMarkup/Commentator.

**Ecosystem context (March 2026):**
- AGENTS.md became a Linux Foundation open standard (20,000+ repos)
- GitHub shipped Spec Kit — markdown-as-programming-language workflow
- Cloudflare launched Markdown for Agents (80% token reduction vs HTML)
- Visual Studio Magazine: "In agentic AI, it's all about the markdown"

**Patterns discovered worth evaluating:**
- **CriticMarkup** — `{~~old~>new~~}` inline suggestions. Accept/reject like Word track changes. Already in Obsidian via Commentator plugin.
- **Confidence indicators** — AI marks `[confidence: high/medium/low]` on recommendations. Human challenges low-confidence items.
- **Dependency tracking** — `(blocked-by: database schema)` on tasks. Reader grays out blocked items.
- **Progress bars** — Auto-calculated from checkbox state: `████████░░ 80% (8/10)`
- **Status state machines** — `draft → review → approved → implemented`. Only valid transitions allowed.
- **Conditional sections** — Show/hide content based on earlier decisions.
- **Priority rankings** — Reorderable priority list with drag-and-drop.
- **Inline discussion threads** — Conversation anchored to document sections.

**Pain points in the wild:**
- Context loss across sessions (biggest complaint)
- No standard for AI-human handoff points in documents
- One-way documents (humans write for AI, or AI writes for humans — never bidirectional)
- Fragile text parsing for task managers
- AI-generated markdown formatting breaks in pipelines
